# HRIV Backend API

Use this skill for backend changes that affect FastAPI routing, SQLAlchemy data
models, schemas, auth dependencies, migrations, or backend tests.

## Start Here

1. Read `../../../AGENTS.md` for repo-wide invariants.
2. Read `references/backend-map.md` for the backend file map and change checklist.
3. For schema changes, read `../../../backend/README.md` and
   `../../../docs/domain-model.md`; Alembic is the source of truth.
4. For endpoint roles or auth behavior, also use `$hriv-access-control`.
5. For image upload, tiling, replacement, metadata, or annotations, also use
   `$hriv-image-workflows`.
6. For admin export/import or `AdminTask`, also use `$hriv-admin-operations`.

## Backend Rules

- Keep routers thin: request validation and auth in routers, durable behavior in
  helpers such as `admin_ops.py`, `processing.py`, `visibility.py`, or services
  already present in `backend/app`.
- Keep Pydantic schemas in `schemas.py` aligned with SQLAlchemy models and API
  response shapes.
- Add or update Alembic migrations with every schema/model change; review
  autogenerated migrations before committing.
- Preserve `metadata_` as the Python attribute for the DB column named
  `metadata`; do not overwrite JSONB metadata with wholesale writes when a merge
  is required.
- Preserve structured logs and request correlation when touching middleware,
  auth, rate limiting, worker, or task code.

## Validation

Use `../../../docs/agent-test-matrix.md` to choose targeted tests. Backend-only
changes normally run:

```bash
cd backend && poetry run pytest <targeted tests>
```

Run the full backend suite before a PR when practical:

```bash
cd backend && poetry run pytest
```
